THE MODEL

A grant becomes a token. The token is all the agent ever holds.

The user grants. AgentAuth mints. Your server checks the badge. Anyone can call the dog off.

03 VERIFY

04 REVOKE

Call it off

revoke() by token, agent, or user.

QUICKSTART

From zero to a scoped token in under a minute.

Copy each block in order. No accounts, no dashboard, no service to run. It's a library.

1

Install

Node 18+. One dependency (jose).

2

Configure once

A 32-byte secret, server-side only.

3

Issue from a grant

After consent, mint a scoped token.

4

Verify on your API

Gate each route by the scopes it needs.

5

Revoke anytime

One token, or every token an agent holds.

// 1 — install

terminal
npm install agentauth
# a secret (>= 32 bytes): node -e "console.log(crypto.randomUUID()+crypto.randomUUID())"

// 2 — configure once (server-side)

auth.ts
import { AgentAuth } from 'agentauth'

export const auth = new AgentAuth({
  secret:   process.env.AGENTAUTH_SECRET!,  // >= 32 bytes
  issuer:   'my-app',
  audience: 'my-api',
})

// 3 — issue a token after consent

grant.ts
const { token } = await auth.issue({
  subject: 'user_123',            // the human
  agent:   'research-bot',         // the agent
  scopes:  ['email:read', 'calendar:read'],
  ttlSeconds: 600,                 // optional, 15-min default
})

// 4 — verify on every protected route

route.ts
import { MissingScopeError } from 'agentauth'

try {
  const claims = await auth.verify(token, {
    requiredScopes: ['email:read'],
  })
} catch (err) {
  if (err instanceof MissingScopeError) deny(403)
}

// 5 — call the dog off

revoke.ts
await auth.revoke(token)                       // one token
await auth.revoke({ agent: 'research-bot' })   // the whole agent
await auth.revoke({ subject: 'user_123' })    // all of a user's
That's the whole API. For production: swap the in-memory store for Redis via RevocationStore, add an onEvent hook for audit, set a kid for key rotation. All in the README.
WHY NOT JUST AN API KEY

Because a key can't be scoped, revoked, or expired.

// SCOPED

Least privilege, enforced

Every token carries explicit scopes. verify rejects anything beyond them, with wildcards (email:*) for grouping. The agent only ever sees what the user approved.

// SHORT LEASH

Small blast radius

15-minute TTL by default. A leaked token is worthless in minutes.

// REVOCABLE

Kill switch

Revoke a token, or every token an agent or user holds, in one call.

// STATELESS

No DB round-trip

Signed JWTs verify locally. Opt into revocation checks only where needed.

// AUDITABLE

Every event, hooked

Structured issued / verified / denied / revoked events for your trail.
