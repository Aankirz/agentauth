server.ts 
import { AgentAuth } from 'agentauth'

const auth = new AgentAuth({ secret: process.env.SECRET })

// user authorizes an agent for one scope
const { token } = await auth.issue({
  subject: 'user_123',
  agent:   'research-bot',
  scopes:  ['email:read'],
})

// your API enforces it in one call
await auth.verify(token, { requiredScopes: ['email:read'] })
The model

A grant becomes a token. The token is all the agent ever holds.

Four moving parts. The user grants, AgentAuth mints, your server verifies, anyone can revoke.

01 · grant

User consents

Approves an agent for specific scopes with a short TTL.

02 · issue

Mint a token

auth.issue() returns a signed, short-lived JWT.

03 · verify

Server enforces

auth.verify() checks signature, expiry, scopes.

04 · revoke

Cut it off

auth.revoke() by token, agent, or user.

Quickstart

From zero to a scoped token in under a minute.

Copy each block in order. No accounts, no dashboard, no service to run — it's a library.

1

Install

Node 18+. The core has one dependency (jose).

2

Configure once

A ≥32-byte secret, server-side only. Set issuer & audience.

3

Issue from a grant

After the user consents, mint a scoped token for the agent.

4

Verify on your API

Gate each route by the scopes it needs. Throws typed errors.

5

Revoke anytime

By token, or "disconnect this agent" for every token it holds.

# 1 — install

terminal
npm install agentauth
# generate a secret: node -e "console.log(crypto.randomUUID()+crypto.randomUUID())"

# 2 — configure (once, server-side)

auth.ts
import { AgentAuth } from 'agentauth'

export const auth = new AgentAuth({
  secret:   process.env.AGENTAUTH_SECRET!,  // >= 32 bytes
  issuer:   'my-app',
  audience: 'my-api',
})

# 3 — issue a token after consent

grant.ts
const { token, expiresAt } = await auth.issue({
  subject: 'user_123',            // the human
  agent:   'research-bot',         // the agent
  scopes:  ['email:read', 'calendar:read'],
  ttlSeconds: 600,                 // optional, 15 min default
})
// hand `token` to the agent

# 4 — verify on every protected route

route.ts
import { MissingScopeError } from 'agentauth'

try {
  const claims = await auth.verify(token, {
    requiredScopes: ['email:read'],
  })
  // claims.sub, claims.agent, claims.scopes
} catch (err) {
  if (err instanceof MissingScopeError) deny(403)
}

# 5 — revoke

revoke.ts
await auth.revoke(token)                       // one token
await auth.revoke({ agent: 'research-bot' })   // disconnect agent
await auth.revoke({ subject: 'user_123' })    // every token for a user
That's the whole API. For production: swap the in-memory revocation store for Redis via the RevocationStore interface, add an onEvent hook for audit, and set a kid for key rotation. All in the README.
Why not just an API key

Because a key can't be scoped, revoked, or expired.

// scoped

Least privilege, enforced

Every token carries explicit scopes. verify rejects anything beyond them, with wildcard support (email:*) for grouping. The agent only ever sees what the user approved.

// short-lived

Small blast radius

15-minute TTL by default. A leaked token is worthless in minutes.

// revocable

Kill switch

Revoke a token, or every token an agent or user holds, in one call.

// stateless

No DB round-trip

Signed JWTs verify locally. Opt into revocation checks only where you need them.

// auditable

Every event, hooked

Structured issued / verified / denied / revoked events for your audit trail.
